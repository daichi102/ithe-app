# 配送・工事会社システム 仕様書

## 目的

配送・工事会社の案件登録、進捗、要確認対応、編集履歴を管理する作業管理システムを作る。
見積書、請求書、売上、粗利などの売上管理機能は別システムで作成し、本システムとはAPI連携する。
ただし、配送・工事・エアコン工事などの作業完了報告書は作業管理システム側で作成できるようにする。
作業管理システムで作成された作業完了報告書は、売上管理システムへAPI連携して反映する。

## 進め方

ユーザーへの質問を1つずつ行い、回答をもとに仕様を確定していく。

## 確定事項

- 配送と工事は同じ案件内では扱わない。
- 配送は配送案件として管理する。
- 工事は工事案件として管理し、エアコン工事・電気工事などの種別を持つ。
- 配送・工事ともに、現場1件を1案件として登録する。
- 配送案件の登録項目は「ご依頼様」と「設置先」を持つ。
- 配送案件の「ご依頼様」には、顧客名・電話番号・住所を登録する。
- 配送案件の「設置先」には、お客様カナ名・名前・住所・電話番号を登録する。
- 配送案件には「設置商品」を登録する。
- 配送案件の「設置商品」には、品名・品番・色・数量・備考を登録する。
- 配送案件の「設置商品」は、1案件に複数登録できる。
- 配送案件には、弊社問い合わせ番号・訪問設置日・注意事項を登録する。
- 配送案件は、原則としてシステムへ手入力せず、メールに添付されたExcelファイルから自動登録する。
- 配送案件のExcel取込は、システムがメールを自動で読み取り、添付Excelを自動取込する方式を希望する。
- 配送案件の自動登録では、メール本文ではなく、メールに添付されたExcelファイルのシート内容を読み取る。
- 配送案件のExcelは、1ファイルにつき1案件として扱う。
- 配送案件のExcelには複数シートが存在し、その中の「アイザ」シートを読み取る。
- 配送案件のメール自動読取は、Outlook / Microsoft 365 を対象とする。
- メール自動読取には Microsoft Graph API の利用を想定する。
- 配送案件メールは、地域別にメールアドレスを分けて受信する方針。
- 地域区分は、関東・関西・九州・東北・行政・北海道・中部を想定する。
- システムは地域別の複数メールボックスを監視できる必要がある。
- 地域別メールアドレスは、各地域ごとの共有メールボックスとして作成する。
- 配送案件の登録方式は、工事案件とは異なる。
- 工事案件はExcel取込ではなく、システム画面から手入力で登録する。
- 工事名（種別）は、弱電・エアコン・リフォームで区分する。

  - 住所
- 設置先

  - 備考

- 弊社問い合わせ番号
- 訪問設置日
- 注意事項

配送案件の登録方式:

- メールにExcelファイルが添付されて届く。
- Excelファイル内に配送案件情報が記載されている。
- 案件情報はメール本文ではなく、添付Excelのシート内容から読み取る。

- 取込方式は、システムがメールを自動で読み取り、添付Excelを自動取込する方式を希望する。

### Q12. 見積書の計算方式

見積書、原価、利益率、見積金額、利益、粗利の管理は、売上管理別システムで扱う。
本システムでは見積書の作成・計算・出力は行わない。

連携方針:

- 作業管理システムは案件情報、作業状態、設置先、設置商品、訪問設置日、作業完了報告書などの作業管理データを保持する。
- 売上管理システムは見積書、請求書、売上、粗利、原価を保持する。
- 両システムはAPIで案件情報と作業完了報告書情報を連携する。

請求書の発行単位は売上管理別システム側で決定する。
本システムは、請求対象になり得る作業完了データをAPI連携できる状態にする。

### Q15. 作業完了報告書の明細項目

作業完了報告書は作業管理システム側で作成する。
配送、工事、エアコン工事では報告書の内容が異なるため、作業種別ごとに入力項目・表示項目を分ける。
作成された作業完了報告書は、売上管理別システムへAPI連携して反映する。

作業完了報告書の作成方針:

- 作業種別ごとに別テンプレートを用意する。
- 初期対象は、配送・弱電・エアコン・リフォームとする。
- 作業完了報告書を保存したら、自動で売上管理システムへAPI連携する。
- 作業チェック表は作業完了報告書とは別に作成・出力できるようにする。
- お客様サイン欄は作業完了報告書ではなく、作業チェック表側に設ける。

- カテゴリ
- 商品
- 内容
- 作業種別
- 作業内容
- 作業完了日
- 料金
- 標準料金参照
- 追加実費
- 締め日
- 集計期間
- 合計金額
- 作業完了状態
- 連携状態

### Q15-2. 作業チェック表

作業チェック表は、作業完了報告書とは別に作成・出力する。

- 作業チェック表には、お客様サイン欄を設ける。
- お客様サインは、画面上に直接サインして取得する。
- 作業チェック表PDFには、画面で取得したお客様サイン画像を表示する。
- 作業チェック表PDFには、サイン日時を表示しない。
- サイン日時はシステム内部に保存する。
- 作業チェック表のテンプレートは、配送用と工事用に分ける。
- エアコン工事と工事系（弱電・リフォーム）は同じ工事用テンプレートを使用する。
- 作業チェック表の入力項目は別途決める。
- 作業チェック表の出力形式はPDFのみとする。
- 作業チェック表は、案件ステータスが作業完了になった後のみ作成できる。
- 作業チェック表は、1案件につき1枚作成する。
- 作業チェック表は、お客様サイン前であれば保存後も編集できる。
- お客様サイン後は編集できない。
- お客様サイン後に修正する場合は、サインを削除してから編集し、再度お客様サインを取得する。
- サイン削除は一般ユーザーも実行できる。
- サイン削除履歴は残さない。
- 作業チェック表のPDF出力は、お客様サイン後のみ可能とする。
- 作業チェック表は売上管理システムへ連携しない。

### Q16. 作業完了報告書の料金マスタ

作業完了報告書を作成するための料金マスタは、作業管理システム側で参照できるようにする。
料金マスタの主管理を作業管理システム側に置くか、売上管理別システムからAPI参照するかは未確定とする。
作業管理システムでは仮料金を計算し、売上管理システム側で正式な金額を確定する。

配送の作業完了報告書の標準料金は以下を使用する。
工事、エアコン工事など配送以外の料金・項目は別途決める。

配送の作業完了報告書では、以下の料金マスタを画面上で選択式にする。
ユーザーはカテゴリ、商品区分、作業内容を選択し、該当する標準料金を仮料金として自動入力する。
階段上げ・階段下ろし、オプションも同じ料金マスタから選択できるようにする。
配送の作業完了報告書には、料金明細行を複数追加できる。
料金明細行の金額は、標準料金を自動入力した後に手入力で変更できる。
仮料金は、手入力で変更された金額を使って計算する。
金額を手入力で変更する場合も、変更理由欄は作らない。
配送の作業完了報告書では、作業写真は扱わない。
配送の作業完了報告書には、作業担当者名を表示する。
作業担当者名は選択タブで登録ユーザー一覧から選択できる。
候補にない担当者は手入力できる。
作業担当者は主担当を1人、補助担当を複数人選択できる。
配送の作業完了報告書では作業時間は入力せず、作業完了日のみ管理する。
複数の料金明細行と追加実費を合計して、作業管理システム側の仮料金を計算する。
配送の料金明細行では数量を入力しない。
1明細行を1件として扱い、仮料金は各明細金額と追加実費の単純合計で計算する。
料金明細行は保存前のみ削除できる。
保存後の料金明細行は削除できず、内容修正として扱う。

## 売上管理システム連携

- 見積書、請求書、売上、粗利、原価は売上管理別システムで扱う。
- 本システムは作業管理に必要な案件情報、作業状態、設置先、設置商品、訪問設置日、作業完了報告書、編集履歴を管理する。
- 作業チェック表は作業完了報告書とは別に作成・出力する。
- 作業完了報告書は作業管理システム側で作成する。
- 作成された作業完了報告書は売上管理システムへAPI連携して反映する。
- 作業完了報告書は、配送・弱電・エアコン・リフォームごとに別テンプレートで作成する。
- 作業完了報告書テンプレートはシステム固定とし、画面から編集できない。
- 作業完了報告書は、保存時に売上管理システムへ自動連携する。
- 作業完了報告書は、保存後も一般ユーザーが編集できる。
- 案件が `売上連携済み` になった後も、作業完了報告書は編集できる。
- 作業完了報告書の編集履歴には、変更前値・変更後値・変更者・変更日時を残す。
- 作業完了報告書の編集履歴の変更日時は日本時間で記録する。
- 作業完了報告書の作成者・編集者は、ユーザーIDとユーザー名で記録する。
- 作業完了報告書を編集して保存した場合も、売上管理システムへ自動連携する。
- 作業完了報告書を編集して再連携した場合、売上管理システム側の同じデータを上書き更新する。
- `売上連携済み` の案件で作業完了報告書を編集して再連携した場合、連携成功後も案件ステータスは `売上連携済み` のままとする。
- 作業完了報告書の料金は、作業管理システムで仮計算し、売上管理システム側で確定する。
- 作業完了報告書はPDFとExcelで出力できる。

- 案件ステータスは、一部自動、一部手動で変更する。
- `取込済み` と `売上連携済み` はシステムが自動で変更する。
- `確認中`、`作業中`、`作業完了`、`要確認` はユーザーが手動で変更する。
- 手動変更できる案件ステータスは、一般ユーザーも変更できる。
- 案件ステータス変更履歴は残さず、最新ステータスのみ管理する。

- 要確認一覧に残った連携失敗データは、一般ユーザーも再連携できる。
- 再連携に成功した場合は、自動で確認済みにする。
- 本システムと売上管理別システムはAPI連携する。
