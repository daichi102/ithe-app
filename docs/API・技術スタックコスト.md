## 前提

この資料は、作業管理システムの機能に沿って、利用予定のAPI、DB、認証、ホスティング、ファイル保存、外部連携、想定コストを整理するためのメモである。

現時点では、ユーザーへのヒアリング回答をもとにしたドラフトであり、未確認項目は今後追記する。

## 現時点の推奨技術スタック

| 領域 | 採用候補 | 理由 |
|---|---|---|
| DB | Supabase Database / PostgreSQL | 案件、チェック表、連携ログ、操作ログを構造化して保存できる。JSONBも使えるため現行SQLiteの`payload`移行もしやすい。 |
| 認証 | Supabase Auth + Microsoft OAuth | Microsoft 365利用と相性が良い。会社アカウントでログインでき、ユーザー管理を統一しやすい。 |
| ファイル保存 | Supabase Storage | 作業完了報告書PDF、チェック表PDFの保存に使う。DBにはStorageパスとメタ情報を保存する。 |
| メール取込 | Microsoft Graph API | Microsoft 365 / Outlookが確定しているため。7個の地域別メールボックスを監視する。 |
| 定期処理 | Supabase Edge Functions | 1時間おきのメール取込、手動取込、外部API連携をサーバー側で安全に実行する。 |
| 画面ホスティング | Vercel | 初期コストが低く、フロント画面の配信に向いている。 |
| PDF生成 | 初期はサーバー側API/Functions、重い場合はRender | 帳票PDF生成は処理が重くなる可能性があるため、負荷を見て配置先を決める。 |
| 売上管理連携 | 売上管理システムAPI | 完了報告書保存時に連携し、失敗時は自動再試行する。 |
| GPT API | 見積候補生成・作業内容判定の補助 | 正式金額や請求確定は売上管理システム側で行う。GPT単体で金額確定しない。 |

|---|---|---|

| Supabase Auth | ログインと権限管理 | 管理者、現場員のログイン管理を行う。Microsoft 365アカウントでログインできるようにする。 |

| Supabase Storage | PDFなどのファイル保存 | このシステムで作成した作業完了報告書PDF、チェック表PDFを保存する。サイン画像は単体保存せずPDFに埋め込む。 |
| Supabase Edge Functions | サーバー側処理、定期処理、外部API連携 | 1時間おきのメール取込、手動メール取込、Microsoft Graph API呼び出し、売上管理API連携、自動再試行を実行する。 |

## 注意点

- 顧客情報、住所、電話番号、サイン入りPDFを扱うため、権限管理、通信暗号化、バックアップ、操作ログは必須とする。
- GPT APIは正式な金額確定や請求確定の責任主体にしない。金額根拠は売上管理システム側の料金マスタに置く。
- 見積書PDFや請求関連PDFは売上管理システム側を正式な保存先とし、このシステムでは送信済みステータスを表示する。
- 作業完了報告書PDFとチェック表PDFはこのシステム側で作成し、Supabase Storageに保存する。
- サイン画像は単体保存せず、PDFに埋め込んで保存する。

### 現時点の推奨予算

初期本番では、月額予算として **15,000円〜20,000円程度** を見ておくのが妥当である。

理由:

- Supabase ProでDB、認証、Storage、Edge Functionsをまとめて開始できる。
- 月150〜200件規模なら、DB容量、Storage容量、API実行回数はいずれも大きくなりにくい。
- Microsoft 365は7メールボックス分を見込むと、月6,000円台になる。
- GPT APIは初期利用量が小さければ、月数千円程度に収まる可能性が高い。
- Vercelは初期は無料枠で開始し、商用チーム運用や制限が問題になった時点でPro化すればよい。

| Microsoft 365を全利用者14人分契約する | 約12,600円/月まで増える |
| Vercel Proを使う | 約3,100円/月増える |
| PDF生成をRender等の別バックエンドに分離する | 約1,000〜3,000円/月以上増える |
| GPT APIで見積作成を大量実行する | 実行回数と入力データ量に応じて増える |
| Storageに写真やPDFを大量保存する | Supabase Storage追加料金が発生する可能性がある |
| 売上管理システムAPIに別料金がある | 連携先の契約・API課金に依存する |

### 結論

このシステムの初期本番コストは、現実的には **月12,000〜15,000円程度** から開始できる見込みである。

ただし、余裕を持った予算取りとしては **月15,000〜20,000円程度** を推奨する。

## 下請け業者に利用してもらう場合のシステム使用料案

### 前提

ここでの対象は、外部の一般企業へSaaSとして販売する相手ではなく、**自社の配送・工事案件を実際に対応してくれる下請け業者** とする。

この場合、システム使用料は利益を大きく取るための販売価格ではなく、以下を目的に設計する。

| 目的 | 内容 |
|---|---|
| 利用定着 | 下請け業者が負担に感じず、確実に使ってくれる金額にする |
| 原価回収 | Supabase、Microsoft 365、Storage、通知、保守の一部を回収する |
| 業務統一 | チェック表、完了報告、サイン、連携ステータスを同じルールで管理する |
| トラブル削減 | 紙、電話、LINE、メールばらばらの報告を減らす |

### 結論

下請け業者から高い月額を取るのはおすすめしない。

推奨は **1社あたり月額3,000〜5,000円**、または **案件単価から1件あたり100〜200円をシステム利用料として差し引く方式** である。

| 方式 | 金額案 | おすすめ度 | 理由 |
|---|---:|---|---|
| 無料提供 | 0円 | 高 | 業務上必須ツールとして使わせやすい。利用率が上がる。 |
| 月額固定 | 3,000〜5,000円/社 | 高 | 負担が小さく、インフラ費の一部を回収できる。 |
| 案件ごと | 100〜200円/件 | 高 | 案件数に応じて公平。月額請求より受け入れられやすい。 |
| 高額月額 | 10,000円以上/社 | 低 | 下請け側の反発が出やすく、利用定着の妨げになる。 |

### 推奨パターン

最も現実的なのは、以下のどちらかである。

| パターン | 内容 | 月額イメージ |
|---|---|---:|
| A. 無料提供 | 自社が全額負担し、下請け業者には必須業務ツールとして使ってもらう | 0円 |
| B. 低額固定 | 下請け1社あたり月額3,000円〜5,000円をシステム利用料として請求 | 3,000〜5,000円/社 |
| C. 案件控除 | 1案件あたり100円〜200円をシステム利用料として差し引く | 月100件なら10,000〜20,000円 |

初期は **A. 無料提供** または **B. 月額3,000円** がよい。

理由:

- 下請け業者にとっては、仕事を受けるために使うシステムであり、負担が大きいと反発が出やすい。
- 自社側には、報告回収、チェック漏れ防止、完了報告の標準化、売上管理連携の効率化というメリットがある。
- システム利用料を高く取るより、利用率を上げて業務を標準化する方が効果が大きい。
- 月3,000円程度なら、下請け側も事務負担削減の対価として受け入れやすい。

### 案件数別の目安

| 下請け業者の月間対応件数 | 月額固定の場合 | 案件ごと100円の場合 | 案件ごと200円の場合 |
|---:|---:|---:|---:|
| 10件/月 | 3,000円 | 1,000円 | 2,000円 |
| 30件/月 | 3,000円 | 3,000円 | 6,000円 |
| 50件/月 | 5,000円 | 5,000円 | 10,000円 |
| 100件/月 | 5,000円 | 10,000円 | 20,000円 |

件数が少ない下請けには月額固定が重く感じられるため、**1件あたり100円** の方が公平である。

一方で、請求処理を簡単にしたい場合は、**月額3,000円/社** から始めるのが扱いやすい。

### おすすめの最終案

最初の運用では、以下を推奨する。

| 区分 | 使用料 |
|---|---:|
| 初期導入期間 | 無料 |
| 通常利用 | 1社あたり月額3,000円 |
| 対応件数が多い業者 | 月額5,000円、または1件100円 |
| 特別な個別対応が必要な業者 | 実費または個別見積 |

### 実務上の注意

- 「システムを売る」のではなく、「当社案件の報告・チェック・完了処理に必要な共通ツール」として位置付ける。
- いきなり有料にすると反発が出やすいため、最初の1〜3か月は無料にする。
- 有料化する場合は、システム利用料という名目より、管理費、事務処理費、報告書電子化費などの名目の方が受け入れられやすい。
- 下請け業者から月額を取る場合でも、自社側の業務効率化メリットが大きいため、利益目的で高く設定しすぎない。

### 結論

下請け業者向けの使用料は、**月額3,000円/社** から始めるのが最も現実的である。

件数に応じて公平にしたい場合は、**1案件あたり100円** をシステム利用料として設定する。

自社の業務統一を最優先するなら、初期は無料提供し、利用が定着してから月額3,000円または1件100円に移行する。
