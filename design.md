# 工事・配送案件管理システム UIデザイン仕様

## 1. デザインコンセプト

このシステムは、Outlookメール・Excel取込・AI解析を使って、工事/配送案件を管理する業務システムである。

デザインの目的は、現場担当者・事務員・管理者が迷わず使えること。
特に「要確認」「未処理」「処理済み」「請求前」など、止まっている作業を一目で分かるようにする。

## 2. 基本方針

- 案件中心設計にする
- クリック数を減らす
- ステータスを色で分かりやすくする
- iPadでも押しやすいUIにする
- 管理者だけが編集できる項目を明確にする
- 重要操作には確認モーダルを出す
- 現場でも事務でも使えるシンプルな業務UIにする

## 3. デザインテーマ

### 全体イメージ

- 業務システム
- SaaS管理画面
- 近未来感は少しだけ
- 清潔感
- 信頼感
- 現場管理に合う堅実なUI

### 参考方向性

- Linearのような整理された管理画面
- Notionより業務寄り
- Salesforceより軽くて分かりやすい
- shadcn/uiのようなモダンUI

│ [                ] │
│ パスワード          │
│ [                ] │
│                    │

### デザイン

## 11. ダッシュボード画面

### 目的

今日やるべきこと、止まっている案件、処理状況を一目で確認する。

### 上部サマリーカード

```text
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│未処理 12│ │要確認 5│ │本日配送 8│ │処理済 32│
└──────┘ └──────┘ └──────┘ └──────┘
```

### 表示内容

- 未処理件数
- 要確認件数
- 本日配送件数
- 処理済み件数
- 未請求件数
- AI取込エラー件数

### メインエリア

- 左: 本日の案件
- 右: 要確認リスト
- 下: 地域別案件数 / ステータス別件数

### UIルール

- 要確認は赤で強調
- 今日対応するものを最上部に表示
- 1クリックで案件詳細へ遷移
- 未確認・未請求は目立たせる

- 処理済みは薄い緑バッジ

## 14. 要確認詳細画面

### 目的

AIやExcel取込で想定外になったデータを人が確認・修正する。

### レイアウト

```text
要確認: AI解析エラー

エラー内容:
型番が判定できませんでした

Excel原文:
「AQR-XXXX?」

候補:
○ AQR-26R2
○ AQR-TXA50R
○ 手入力

修正入力:
[              ]

[保存] [確認済みにする]
```

### UIルール

- 赤・黄色を使って注意感を出す
- エラー箇所を目立たせる
- 修正前 / 修正後を並べる
- 管理者のみ「確認済みにする」可能
- 確認済みにしたら案件ステータスを処理中または処理済みに変更

### デザイン

- 丸みのあるバッジ

例:

```text
[要確認]

```

## 18. モーダル仕様

重要操作では確認モーダルを出す。

### 対象操作

- 案件削除
- 確認済みに変更
- 請求書発行
- Outlook連携解除
- 管理者権限変更

### モーダル例

```text
この案件を確認済みにしますか？

この操作は履歴に記録されます。

[キャンセル] [確認済みにする]
```

## 19. 入力フォーム仕様

- ラベルは必ず表示する
- 必須項目には「必須」バッジ
- エラーは赤文字
- 入力欄は高さ44px以上
- iPadで押しやすくする

## 20. テーブル仕様

- ヘッダー固定
- 行クリックで詳細へ遷移
- ステータスは左側
- 操作ボタンは右側
- 10件 / 25件 / 50件で表示切替
- 検索結果がない場合は空状態UIを表示

## 21. 空状態UI

データがない場合は、ただ空白にしない。

例:

```text
案件がありません
ExcelまたはOutlookから案件を取り込むとここに表示されます。
```

## 22. エラー表示

### 通常エラー

画面上部にトースト表示。

```text
保存に失敗しました。もう一度お試しください。
```

### 重要エラー

カード内で赤表示。

```text
Excelの型番が読み取れませんでした。
要確認画面で修正してください。
```

## 28. 最重要UXルール

- 要確認は必ず目立たせる
- 案件詳細からすべての処理に進める
- 1案件1画面で完結する
- 管理者だけ編集できることを明確にする
- 操作履歴を必ず残す
- 検索とフィルターを強くする
- 現場で迷わない文言にする

## 30. 最終的な理想

このシステムは「案件詳細」を中心に、メール取込から請求・完了報告までを一気通貫で管理できる業務システムにする。

現場担当者は迷わず作業でき、事務員は未処理・要確認をすぐ見つけられ、管理者は数字と履歴を正確に追える状態を目指す。
